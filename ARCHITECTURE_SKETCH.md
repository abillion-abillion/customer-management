# 고객 재무관리 시스템 스케치

## 1) 목표
- 고객별 재무제표(엑셀)와 투자 포트폴리오(PDF/PNG)를 한 곳에서 누적 관리
- 고객 본인 로그인 후 본인 데이터만 상시 조회
- 시점별 로그를 바탕으로 변화(개선/악화), SWOT, 칭찬/개선 포인트를 시각화

## 2) 화면 구조
- `/` 로그인
- `/admin` 관리자 대시보드
- 엑셀 업로드(스냅샷 생성)
- 고객 검색(이름/라벨 예: `95_여_권태희`)
- 고객 목록 클릭 진입
- `/customer/[id]` 고객 상세
- KPI 카드(소득/지출/저축/총자산/순자산/총자산 수익률)
- 추세 그래프(소득, 지출, 순자산, 총자산, 수익률)
- SWOT + 칭찬/개선 포인트
- 금융자산/부채/보험/부동산 표
- 포트폴리오 파일(PDF/PNG) 뷰어 및 관리자 업로드

## 3) 사용자 흐름
1. 관리자 로그인
2. 재무제표 업로드 + 스냅샷 라벨 선택(컨설팅 전/후, n차 점검 등)
3. 고객 자동 생성/갱신 + 고객 계정 자동 생성
4. 고객 검색 후 상세 페이지 진입
5. 필요 시 포트폴리오 파일(PDF/PNG) 업로드
6. 고객은 본인 계정으로 로그인 후 본인 페이지만 조회

## 4) 백엔드/API 스케치
- `POST /api/auth` 로그인
- `GET /api/customers?q=` 고객 검색
- `POST /api/upload` 엑셀 업로드/파싱/DB 저장
- `GET /api/customers/[id]` 고객 상세 + 최신 상세 테이블 + 포트폴리오 파일 목록
- `GET /api/analysis/[id]` SWOT, 추세, 칭찬/개선 인사이트
- `POST /api/portfolio/[id]` 포트폴리오 파일 업로드(관리자 전용)

## 5) 데이터 모델 핵심
- `customers`: 고객 기본 정보
- `snapshots`: 시점별 요약 지표(로그)
- `assets`, `debts`, `insurance`, `real_estate`, `expense_details`: 상세 행 데이터
- `portfolio_files`: 고객별 PDF/PNG 문서 저장
- `users`: 관리자/고객 인증 정보

## 6) 분석 로직 스케치
- 스냅샷 최근 2개 이상 비교
- 개선/악화 판단: 소득, 지출, 부채, 순자산, 총자산 수익률
- 현재 상태 평가: 저축률, 부채비율, 월상환 부담률
- 결과 출력: SWOT 4분면 + 칭찬/개선 포인트 + 추세 그래프

## 7) 운영 포인트
- 고객 100명+ 기준: 이름/라벨 검색 중심 운영
- 파일 형식: 재무제표 `xlsx`, 포트폴리오 `pdf/png/jpg/webp/gif`
- 보안: 고객은 본인 `customer_id` 데이터만 접근 가능

## 8) 포트폴리오 운영 규칙(제안)
- 업로드 기본 연결: 최신 스냅샷(점검 이력)에 자동 연결
- 파일 메모 규칙: `운용보고`, `리밸런싱`, `성과캡처`, `상담자료` 중 하나 + 짧은 설명
- 월 1회 이상 업데이트가 없으면 다음 상담 전 점검 알림
- 조회 우선순위: 최신순, 스냅샷 라벨별 필터, 파일명 검색
